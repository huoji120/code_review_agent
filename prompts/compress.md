# 上下文压缩

请把当前对话压缩成一份可以继续代码审计的中文摘要。压缩结果会替代完整历史对话，所以必须保留恢复审计所需的全部关键信息。

必须保留：

- 用户的审计目标和目标目录。
- 当前审计计划。
- todo 状态。todo 必须具体，包含文件、目录、模块、函数、变量或安全审计点。
- 文件排查状态：未看、正在审计、已审计、跳过，以及重要文件 note。
- 变量/函数/符号排查状态，以及关键数据流 note。
- 跨文件调用链/数据流 flow 状态，包括入口、经过文件、关键变量、已确认检查、缺失检查、sink、证据和下一步。
- 接下来应该创建或更新的详细 todo。不能写“继续审计”这类空泛 todo。
- 已检查文件和相关行号范围。
- 已提交漏洞及其证据。
- 未解决的问题。
- 下一步具体动作。
- 后续配置轮数计划：按系统配置的 summary interval，规划后续 N 轮每轮优先做什么。
- 连续失败信息：如果某个怀疑点、变量、入口、注解、权限链或利用链已经连续20轮都没有找到新证据、没有闭合、没有新增命中或没有推进，必须在摘要中明确标记为“连续20轮无证据”，并要求恢复后停止继续在同一方向空转，改换新线索或新入口。

应该丢弃：

- 已经不重要的工具长输出。
- 重复讨论。
- 无关日志。

输出必须使用中文，并按下面格式：

## 审计摘要

## Todo 状态

## 文件排查状态

## 变量排查状态

## 跨文件 Flow 状态

## 已提交漏洞

## 已检查文件

## 未解决问题

## 下一步

## 后续 N 轮计划

在 `下一步` 中明确说明 agent 恢复后应该马上做什么，并列出应该调用 `todo_create` 或 `todo_update` 写入的详细 todo。todo 必须包含具体文件/模块/入口/变量/审计点。若存在未闭合的跨文件链路，必须说明下一步要读哪个文件、追踪哪个函数/变量，并更新对应 flow。在 `后续 N 轮计划` 中，N 使用本次压缩请求提供的 summary interval，逐轮列出接下来 N 轮的审计计划。

如果压缩内容显示某个方向已经连续20轮没有任何新证据、没有新命中、没有推进或只是重复搜索同一类关键词，那么 `下一步` 和 `后续 N 轮计划` 里不允许继续重复同一动作，必须切换到新的文件、入口、变量、权限边界或利用链方向。
